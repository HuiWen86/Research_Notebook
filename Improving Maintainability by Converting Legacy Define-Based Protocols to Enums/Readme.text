프론트 메인 pc 로 이어지는 통신 프로토콜 구조 개선 (define 에서 enum 기반 으로 개선) 
보조: 기존 define 프로토콜의 enum 전환을 통한 유지보수성 향상 연구 

1목적 : 프로젝트에서 사용중인 다수의 define 기반 프로토콜 상수를 enum 기반 구조로 재정비하여 
코드의 가독성, 유지보수성, 안정성을 향상시키고, 향후기능 확장시 발생 가능한 충돌을 예방.

2배경 : 현재 front.c apps.c 펌웨어 프로토콜 식별자를 define 상수로 관리 중임. 하지만 
define방식은 코드가 늘어날 수록 수정과 추적이 어렵고 상수 그룹화가 되지 않으며 중복값이 발생해도 
컴파일러 경고가 없고 디버깅시 심볼명이 보이지 않음. 

3문제점분석 :
  ㄱ. 관리 포인트 증가 : define 은 파일 어디에서나 선언될 수 있어 추적이 어려움. 특정 프로토콜이 어디에 속하는지 확인 어려움. 
  ㄴ. 중복 선언과 충돌 위험있음. 
  ㄷ. 타입에 안정성이없음 : define 은 단순 치환이라 데이터 타입 보호가 안되고 잘못된 값이 대입되어도 체크 안됨. 
  ㄹ. 디버깅 어려움 
  ㅁ. 가독성 문제 : 프로토콜 구조가 한눈에 파악 어려움. 

  4개선전략 : 프로토콜 사용목적에 따른 분류, 기존의 define 을 유지하면서 차츰 enum 을 도입, 구조체 기반 소스를 만들어 사용 

  5기대효과  :
    ㄱ. 유지 보수성 향상
    ㄴ. 안정성 증가 
    ㄷ. 디버깅 효율 상승
    ㄹ. 코드 일관성 확보 및 확장성
